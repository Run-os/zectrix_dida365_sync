# Zectrix DIDA365 同步计划（更新版）

## 1. 数据模型映射

### DIDA365 与 Zectrix 字段映射

| DIDA365 字段   | Zectrix 字段           | 映射说明                                                                |
|----------------|------------------------|-------------------------------------------------------------------------|
| `title`        | `title`                | 直接映射                                                                |
| `content`      | `description`          | 直接映射，同时在描述末尾添加DIDA365任务ID标记                           |
| `dueDate`      | `dueDate`              | DIDA365的ISO 8601格式转换为 `yyyy-MM-dd`                                |
| `dueDate`      | `dueTime`              | 从DIDA365的ISO 8601格式中提取时间部分，转换为 `HH:mm`                   |
| `priority`     | `priority`             | DIDA365的优先级（0/1/3/5）映射到Zectrix的优先级（0/1/2）                |
| `status`       | `status` / `completed` | DIDA365的状态（0/2）映射到Zectrix的状态（0/1）和completed（false/true） |
| `id`           | `description`          | 存储在Zectrix的description字段中，格式为 `[DIDA365:{task_id}]`          |
| `modifiedTime` | 同步记录               | 用于冲突检测                                                            |

### ID 映射方案

由于Zectrix没有 `externalId`字段，我们采用以下方案：

1. **从 DIDA365 到 Zectrix**：

   - 在创建Zectrix待办事项时，在 `description`字段末尾添加DIDA365任务ID标记
   - 格式：`[DIDA365:{task_id}]`
   - 例如：`买牛奶 [DIDA365:69da2a94e4b075626889b602]`
2. **从 Zectrix 到 DIDA365**：

   - 解析Zectrix待办事项的 `description`字段，提取DIDA365任务ID
   - 如果找到ID标记，则更新对应DIDA365任务
   - 如果未找到ID标记，则创建新的DIDA365任务

## 2. 同步策略

### 2.1 双向同步逻辑

1. **从 DIDA365 到 Zectrix**：

   - 获取DIDA365项目列表
   - 选择指定项目的任务
   - 将任务同步到Zectrix设备
   - 在Zectrix的 `description`字段中添加DIDA365任务ID标记
2. **从 Zectrix 到 DIDA365**：

   - 获取Zectrix设备的待办事项
   - 解析 `description`字段，提取DIDA365任务ID
   - 将待办事项同步到DIDA365指定项目

### 2.2 冲突解决机制

- **基于最后修改时间**：当两边都有修改时，保留最后修改的版本
- **同步标记**：为每个同步项添加同步时间戳和来源标记
- **手动解决**：对于复杂冲突，记录冲突并提示用户手动解决

### 2.3 同步范围

- **同步对象**：仅同步活跃的待办事项（未完成状态）
- **同步频率**：可配置，默认每5分钟同步一次
- **同步方向**：默认双向同步，可配置为单向

## 3. 同步流程

### 3.1 初始化同步（全量同步）

1. **准备阶段**：

   - 加载环境变量和配置
   - 初始化API客户端
   - 检查网络连接和API可用性
2. **从 DIDA365 同步到 Zectrix**：

   - 获取DIDA365项目列表
   - 选择目标项目
   - 获取项目下所有任务
   - 对每个任务：
     - 检查Zectrix中是否已存在对应任务（通过解析 `description`字段中的DIDA365任务ID）
     - 不存在则创建，存在则更新
     - 在 `description`字段末尾添加DIDA365任务ID标记
3. **从 Zectrix 同步到 DIDA365**：

   - 获取Zectrix设备的待办事项
   - 对每个待办事项：
     - 解析 `description`字段，提取DIDA365任务ID
     - 检查DIDA365中是否已存在对应任务
     - 不存在则创建，存在则更新
4. **完成阶段**：

   - 记录同步状态和统计信息
   - 清理临时数据

### 3.2 增量同步

1. **准备阶段**：

   - 加载上次同步时间
   - 初始化API客户端
2. **从 DIDA365 同步到 Zectrix**：

   - 获取上次同步时间后修改的任务
   - 对每个任务：
     - 检查Zectrix中是否已存在对应任务（通过解析 `description`字段中的DIDA365任务ID）
     - 不存在则创建，存在则更新
     - 如果任务已完成，在Zectrix中标记完成
     - 在 `description`字段末尾添加DIDA365任务ID标记
3. **从 Zectrix 同步到 DIDA365**：

   - 获取上次同步时间后修改的待办事项
   - 对每个待办事项：
     - 解析 `description`字段，提取DIDA365任务ID
     - 检查DIDA365中是否已存在对应任务
     - 不存在则创建，存在则更新
     - 如果待办事项已完成，在DIDA365中标记完成
4. **完成阶段**：

   - 更新上次同步时间
   - 记录同步状态和统计信息

## 4. 错误处理与重试机制

### 4.1 错误类型与处理

| 错误类型       | 处理策略                                 |
|----------------|------------------------------------------|
| 网络错误       | 重试3次，每次间隔递增（1s, 3s, 5s）      |
| API错误（4xx） | 记录错误并跳过该项目，继续同步其他项目   |
| API错误（5xx） | 重试3次，每次间隔递增（1s, 3s, 5s）      |
| 数据格式错误   | 记录错误并跳过该项目，继续同步其他项目   |
| 认证错误       | 记录错误并停止同步，提示用户更新认证信息 |

### 4.2 重试机制

- **指数退避策略**：重试间隔逐渐增加
- **最大重试次数**：3次
- **失败处理**：超过最大重试次数后，记录错误并跳过该操作
- **恢复机制**：下次同步时重新尝试失败的操作

## 5. 定时同步实现

### 5.1 实现方式

- **使用 APScheduler**：设置定时任务，支持cron表达式
- **同步频率**：默认每5分钟一次，可通过配置修改
- **启动方式**：
  - 作为后台服务运行
  - 支持手动触发同步

### 5.2 配置选项

| 配置项            | 说明                                                                                    | 默认值        |
|-------------------|-----------------------------------------------------------------------------------------|---------------|
| `SYNC_INTERVAL`   | 同步间隔（分钟）                                                                        | 5             |
| `SYNC_DIRECTION`  | 同步方向（bidirectional/unidirectional_dida_to_zectrix/unidirectional_zectrix_to_dida） | bidirectional |
| `DIDA_PROJECT_ID` | DIDA365目标项目ID                                                                       | 收集箱        |
| `SYNC_COMPLETED`  | 是否同步已完成任务                                                                      | false         |

## 6. 同步状态记录与追踪

### 6.1 日志记录

- **日志级别**：INFO、WARNING、ERROR

- **日志存储**：
  - 本地文件日志
  - 可选：远程日志服务

### 6.2 同步状态存储

- **存储方式**：本地JSON文件
- **存储内容**：
  - 上次同步时间
  - 同步统计信息
  - 失败任务记录
  - 冲突记录
- **状态恢复**：重启后从上次同步状态继续

### 6.3 监控机制

- **健康检查**：定期检查同步服务状态
- **告警机制**：连续失败多次时发送告警
- **监控指标**：
  - 同步成功率
  - 同步延迟
  - 错误率
